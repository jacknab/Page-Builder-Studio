You are building a production-ready AI website builder SaaS platform (similar to Wix/Squarespace but simplified).

The system generates website templates using OpenAI and renders them dynamically with controlled image selection.

---

# 🎯 CORE GOAL

Build a system where:

* AI generates structured website templates (NOT raw final images)
* Templates are stored in a database
* Images are automatically selected based on section type + business context
* Websites are fully responsive and SEO optimized
* No random or irrelevant images are ever used

---

# 🧱 TECH STACK

* Node.js (Express)
* SQLite (or PostgreSQL)
* OpenAI API (GPT-4o-mini or similar)
* Unsplash API (for images)
* Vanilla HTML/CSS frontend (no heavy frameworks)

---

# ⚙️ CORE ARCHITECTURE

The system must follow this pipeline:

1. User requests template generation
2. OpenAI generates structured JSON template (NOT HTML images)
3. Backend resolves images using controlled image mapping system
4. Final HTML is assembled and stored in database
5. Site is rendered dynamically based on stored template

---

# 🤖 OPENAI OUTPUT FORMAT (STRICT)

OpenAI MUST return ONLY JSON in this structure:

{
"name": "Luxury Nail Salon",
"type": "nail-salon",
"style": "luxury",
"seo": {
"title": "Luxury Nail Salon in [City]",
"description": "High-end nail salon offering premium manicure and pedicure services."
},
"sections": [
{
"type": "hero",
"headline": "Luxury Nails, Redefined",
"subtext": "Premium nail care in a modern spa experience.",
"image_intent": "luxury nail salon interior pink aesthetic"
},
{
"type": "services",
"items": ["Gel Manicure", "Acrylic Nails", "Nail Art"],
"image_intent": "nail technician applying gel polish close up"
},
{
"type": "gallery",
"image_intent": "creative nail art macro photography pastel designs"
},
{
"type": "contact",
"image_intent": "modern spa reception clean minimal aesthetic"
}
]
}

NO HTML IS GENERATED BY OPENAI. ONLY JSON.

---

# ⚙️ IMAGE RESOLUTION LOGIC

For each section:

1. Read section.type
2. Read section.image_intent
3. Map to IMAGE MAP category
4. Select random keyword from that category
5. Fetch image from Unsplash API using keyword
6. Replace section image with real URL

NEVER allow AI to directly output image URLs.

---

# 🌐 UNSPLASH INTEGRATION

Use:

GET https://api.unsplash.com/search/photos?query={keyword}

Return:

* first result image URL
* fallback image if none found

Build a renderer that:
